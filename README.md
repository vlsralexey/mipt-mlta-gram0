# Интерпретатор грамматики класса ноль

## Описание

**Грамматика класса ноль** - это набор из $\Sigma$, $Г$, $P$, $S$,
где $\Sigma$ - алфавит букв, из которых состоят слова,
$Г$ - технические символы (нетерминалы), $P$ - правила вида $\alpha \rightarrow \beta$.

**Алгоритм.** Начинаем со стартового символа $S$ (строки из одного символа).
Далее применяем правила вида $\alpha \rightarrow \beta$, пока это возможно:
если в слове $x$ есть подстрока $\alpha$, заменяем её на $\beta$.

Вывод считается оконченным, когда в слове не остаётся нетерминальных символов.
Для этого есть ограничение на правила: в `\alpha` должен быть хотя бы один
нетерминальный символ.

Говорим, что грамматика **вычисляет функцию $f(x)$**,
если в ней из $BxE$ можно вывести $Bf(x)E$,
где $B$, $E$ - технические нетерминалы для маркировки начала и конца строки.
Будем считать, что вывод вычисление функции $f(x)$ окончено, когда слово
имеет вид $B...E$, а между $B$ и $E$ не осталось нетерминальных символов или
не применимо ни одно из правил.

Грамматики **не детерминированы**, однако для простоты интерпретатор
будет выбирать **первое** применимое правило, а затем заменять **первое**
вхождение $\alpha$ на $\beta$.

## Синтаксис

Каждый символ обозначается строкой, состоящей из одного или более символов:

**Нетерминальные символы:** `:<S>`\
**Терминальные символы:** `<S>`

Где `<S>` - строка, состоящая из строчных и заглавных латинских букв, цифр и символов `_`, `.`, `'`.

На вход интерпретатору подаётся файл с правилами, каждая строка которого имеет вид:
```
a_1,a_2,...,a_n -> b_1,b_2,...,b_n
```
Где `a_1`, `a_2`, ..., `a_n` и `b_1`, `b_2`, ..., `b_n` - символы.

Символы $B$ и $E$ обознаются как `:B` и `:E`.
